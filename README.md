## What Is This

Springwell-Keel is a greenfield, open-source, agentic workflow orchestration framework powering three independent SaaS products from a single intelligent core.

One agentic engine — the **Keel Engine** — powers three completely separate product lanes simultaneously:

| Lane | Purpose |
|------|---------|
| Lane A | Individual Journey Orchestration |
| Lane B | Organizational Service Catalog |
| Lane C | Capability Sequencing & Readiness Handoff |

We are building in public, from day zero. There is no legacy codebase. No inherited architecture. No patterns already baked in.

If you have been waiting for a project where you can shape the bones of a real agentic system before they calcify — **the window is now.**

---

## Architecture

Springwell-Keel is organized into distinct subsystems:

- **Workflow Engine** — orchestration layer (public)
- **Reference Lane Environments** — A, B, C implementations (public)
- **Keel Engine** — judgment layer, scoring models, routing logic (private, maintained by core team)

The public chassis is open source. The internal reasoning layer is deliberately kept private. This boundary is documented from day zero and will not change.
